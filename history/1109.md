- **질문**: 로컬 서버 주소가 `http://localhost:5173/front_7th_chapter2-1/`로 바뀐 이유는?  
  **답변**: GitHub Pages 배포를 위해 `vite.config.js`에 `base: "/front_7th_chapter2-1/"`를 추가해 개발 서버 경로도 동일하게 맞췄다고 설명.

- **질문**: 이전에는 `http://localhost:5173/`에서 화면이 보였는데 지금은 왜 `http://localhost:5173/front_7th_chapter2-1/`에서만 보이나?  
  **답변**: `base` 설정으로 자원 경로가 `/front_7th_chapter2-1/` 아래로 이동해 루트에서는 정적 파일을 찾지 못한다고 안내.

- **질문**: 원격 저장소에 올린 뒤 `pnpm install`에서 오류가 나는 이유는?  
  **답변**: 로컬에서는 정상 설치되며, 원격에서 문제라면 Node/Pnpm 버전, 캐시, `package.json`/`pnpm-lock.yaml` 반영 여부를 점검하라고 제안.

- **질문**: `Ctrl + Shift + L`을 같은 글자 모두 선택 기능으로 바꾸려면 어떻게 해야 하나?  
  **답변**: Cursor/VS Code의 `Keyboard Shortcuts` 혹은 `keybindings.json`에서 `Select All Occurrences of Current Selection`에 원하는 키를 지정하라고 안내.

- **질문**: 배포했는데 GitHub Pages(`gh-pages`)에서 화면이 안 나오는 이유는?  
  **답변**: `base`/`homepage` 설정, `pnpm build && pnpm deploy` 재실행, Pages 브랜치·폴더 설정, SPA용 `404.html` 등을 확인하는 체크리스트를 제공.

- **질문**: `vite.config.js`의 `base`가 주석 처리돼 있어도 페이지가 안 나오는 이유는?  
  **답변**: `base`를 끄면 빌드 자산 경로가 `/`로 고정돼 Pages 경로와 mismatch가 생기므로 배포 시에는 반드시 설정하거나 모드별로 분기하라고 설명.

- **질문**: `pnpm run deploy`를 직접 실행했지만 페이지가 비어 있는 이유는?  
  **답변**: 명령은 성공적으로 `gh-pages`에 배포됐고, 노출 문제는 Pages 설정·캐시 등 다른 요인일 수 있다고 결과를 공유.

- **질문**: 원격 저장소(`remote`)를 다시 설정해야 하나?  
  **답변**: 이미 `origin`과 `upstream`이 등록돼 있으니 추가 설정 없이 `git push origin ...` 등을 그대로 사용하면 된다고 답변.

- **질문**: 전체 파일 줄바꿈을 CRLF에서 LF로 바꿔달라면 어떻게 했나?  
  **답변**: 루트에 `.gitattributes`를 추가해 `* text=auto eol=lf`로 설정하고 `git add --renormalize .`를 실행해 추적 중인 텍스트 파일을 모두 LF로 맞췄다고 보고.

- **질문**: `pnpm run deploy` 후에도 GitHub Pages에 변경 사항이 반영되지 않는 이유는?  
  **답변**: 배포는 정상 수행됐지만, 프로덕션에서도 `msw` 서비스 워커를 강제 실행해 `/mockServiceWorker.js`를 요청하면서 404 → 초기화 미실행으로 빈 화면이 발생하므로, 프로덕션에서는 mocking을 끄거나 서비스 워커 URL을 `"/front_7th_chapter2-1/mockServiceWorker.js"`로 맞추라고 안내.

- **질문**: 배포 페이지가 비어 있는 문제를 해결하기 위해 어떤 수정을 했나?  
  **답변**: `src/main.js`에서 `enableMocking`이 개발 모드에서만 MSW를 시작하도록 분기시키고, 프로덕션과 테스트 모드에서는 바로 `main()`을 실행해 렌더링이 보장되도록 수정했다고 설명.

- **질문**: 로컬 서버 주소가 `http://localhost:5173/front_7th_chapter2-1/`로 바뀐 이유는?  
  **답변**: GitHub Pages 배포를 위해 `vite.config.js`에 `base: "/front_7th_chapter2-1/"`를 추가해서 개발 서버도 동일한 경로를 사용하게 되었음을 설명.

- **질문**: 이전에는 `http://localhost:5173/`에서 화면이 보였는데, 지금은 `http://localhost:5173/front_7th_chapter2-1/`에서만 보이는 이유는?  
  **답변**: `base` 설정 변경으로 자원 경로가 `/front_7th_chapter2-1/` 아래로 바뀌어, 루트 경로에서는 정적 파일을 찾지 못해 화면이 나오지 않는다고 설명.

- **질문**: 원격 저장소에 올린 뒤 `pnpm install`에서 오류가 발생하는 이유는?  
  **답변**: 현재 로컬에서는 `pnpm install`이 정상 동작하며, 문제가 생긴다면 Node/Pnpm 버전 차이, 캐시 문제, 혹은 수정된 `package.json`/`pnpm-lock.yaml`이 원격에 반영되지 않은 경우를 확인하라고 안내.
